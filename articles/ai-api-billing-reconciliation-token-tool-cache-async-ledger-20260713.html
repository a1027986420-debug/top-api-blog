AI API 计费核对怎么做：Token、工具调用、缓存折扣与异步任务要建立同一账本

AI 调用成本并不只有输入和输出 Token。缓存、工具、图像、音频、异步任务与重试都可能造成账单差异。本文给出统一对账方法。

AI API 的账单差异常被归咎于“模型价格变了”，但更多时候是计量口径不一致：业务日志只记聊天 Token，供应商还计入缓存读写、图像像素、音频分钟、工具执行或异步任务轮询。没有统一账本，团队既无法解释成本，也很难判断重试和回退是否真的划算。

为每次调用建立成本事件链

用一个请求 ID 串联客户端请求、中转站路由、上游响应、工具步骤和最终账单项。记录模型版本、输入输出计量、缓存命中类型、图像或音频单位、异步任务状态、重试次数和货币单位。原始账单与内部估算分开保存，避免将预估价格误认为已结算金额。

缓存折扣需要单独验证

不同供应商对缓存读、写和过期的计费方式不同。测试同一提示词首次调用、短间隔复用、超时后复用和只改一个参数四种情况，比较响应中的 usage 与账单导出。若无法得到明细，不要基于“应该命中缓存”给业务方承诺固定节省比例。

异步与工具调用最容易双算

视频、批处理、检索和浏览器工具常有创建费、运行费和结果下载费。失败重试是否免费、取消后是否结算、回调重复是否产生重复任务，都必须写进对账规则。对于可产生写操作的工具，成本和业务副作用也要一起核对，不能只看金额。

中转站的作用是对齐口径

接入 top-api.cc 时，可将多供应商 usage 归一为内部成本事件，并保留原始字段以便审计。归一化不是抹平差异，而是让团队能同时看到“供应商如何收费”和“业务如何使用”。设置预算提醒时，也应按预计、已确认和待结算三类金额分别触发。

月度对账清单

  • 抽样比对内部请求数与供应商账单行；
  • 按模型、租户、项目和任务类型拆分差异；
  • 单独检查缓存、异步取消、重试和工具调用；
  • 对无法映射的费用保留原因码，不强行均摊；
  • 价格与计量规则变更后重跑历史估算。

能解释每一笔成本，才能合理比较模型和工具。统一账本的目标不是生成更漂亮的报表，而是让优化和预算决策有可复核的事实基础。
